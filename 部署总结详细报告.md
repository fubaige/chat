# 一、 系统整理与修改详报

系统目前已完成了对冗余代码结构、安全配置和云端部署参数的前期校验：
1. **结构清理**：基于当前工作目录，发现了根目录与子目录同时存在的 `chat` 目录。我们已将内层冗余备份的大体积 `chat` 文件夹彻底删除。同时也核算保留了业务可能拓展所需的 `wx-mp-svr-main`。
2. **日志调试优化**：在后端的 `Dockerfile` 中新增了 `ENV PYTHONUNBUFFERED=1`，以确保在未来云端监控或浏览器调取系统执行记录时，大模型处理相关日志能够实时暴露而不被缓冲抑制。
3. **敏感信息拦截**：为即将进行的 GitHub 托管和部署同步，优化了 `.gitignore`。全面覆盖了临时进程隔离 (`app.pid`)、上传文件/业务数据池 (`uploads/`, `data/`) 以及关键连接鉴权文件 (`.ssh_deploy_key`)。
4. **前端资源修复**：修正了 `.gitignore` 的泛匹配规则，确保 `static/dist` 编译产物能正确打入 Docker 镜像，彻底解决了 `Internal Server Error` (500) 报错。
5. **项目初始文档**：新建了包含具体连接 IP、端口的完整版 `README.md` 文档。

建议在操作端即刻将当前更改变动同步：
```bash
git add .
git commit -m "chore: 优化目录结构并配置Docker输出及安全策略"
git push origin main
```

## 第 2 步：登录服务器
使用本地保存的 `.ssh_deploy_key`，以管理员身份连入：
*(注意：若提示权限 `Permissions 0644 for '.ssh_deploy_key' are too open`，在 Linux/OSX 需提前执行 `chmod 400 .ssh_deploy_key`。)*
```bash
ssh -i .ssh_deploy_key -p 59582 root@103.36.221.102
```

## 第 3 步：更新服务器代码与重建构建
此步骤在服务器内完成：
```bash
# 进入生产挂载目录
cd /www/wwwroot/chat.aigcqun.cn

# 拉取更新代码
git pull origin main

# 清理过期的冗余Docker构建缓存储备
docker system prune -f

# 使用 docker-compose 进行重新编译和挂起
docker-compose up -d --build
```
